# 관리자 시스템 설계 판단 기록

## [2026-02-23] L3 광역 관리자 알림 기능 불필요 판단

### 결정

L3(광역 관리자) 대시보드에 실시간 알림(벨 아이콘, 폴링)은 **구현하지 않는다.**

### 근거

1. **역할 분리**: L3의 역할은 전략적 감독이며, 개별 환자 대응은 L2의 책임
2. **중복 회피**: L3가 환자 단위 알림을 받으면 L2와 역할이 겹침
3. **대응 불가**: L3가 "고위험 환자 3건" 알림을 받아도 직접 개입할 수단이 없음
4. **대안 존재**: 대시보드 보건소별 현황 테이블에서 고위험 수치가 이미 표시됨

### 향후 고려사항

- L3에 알림을 준다면 환자 단위가 아닌 **센터 단위 이상 징후**여야 의미 있음
  - 예: "A 보건소가 7일간 활동 없음", "B 보건소 고위험 비율 50% 초과"
- 이는 현재 MVP 단계가 아닌 고도화 시점에서 검토

### 현재 알림 체계

| Level     | 알림 | 방식                   |
| --------- | ---- | ---------------------- |
| L4 중앙   | ✅   | 벨 아이콘 + 30초 폴링  |
| L3 광역   | ❌   | 대시보드 통계로 대체   |
| L2 보건소 | ✅   | 실시간 모니터링 테이블 |
