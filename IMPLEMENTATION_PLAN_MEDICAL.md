# 📋 신규 기능 기획 및 서비스 적용 계획서 (약물 관리 및 위기 대응 시스템)

## 1. 개요

본 문서는 '마음온(haruON)' 서비스의 **의료적 효용성(Clinical Utility)을 강화**하고, **사용자 맞춤형 정밀 케어**를 제공하기 위해 제안된 신규 기능들의 상세 기획 및 적용 계획을 다룹니다.

## 2. 주요 신규 기능

### 💊 약물 복용 관리 (Medication Adherence)

약물 복용은 정신건강 치료의 핵심 요소로, 기분 변화와의 상관관계를 시각화하여 의료진과 환자가 함께 모니터링할 수 있도록 합니다.

#### 1. 약물 정보 등록 (Initial Setup)

- **대상**: 약물 치료 중인 사용자
- **입력 정보**: 약 이름, 복용 시간(아침/점심/저녁/취침 전), 용량
- **UI**: 회원가입 시 또는 설정 페이지에서 '복용 중인 약 등록하기' 기능을 제공하여 간편하게 스케줄 설정

#### 2. 일일 복용 체크 (Daily Check-in)

- **기능**: 설정된 스케줄에 맞춰 메인 화면이나 일기 작성 전/후에 '오늘 약 드셨나요?' 체크 박스 노출
- **검증**: 사용자 체크 + (선택 사항) 의료진이 처방 데이터와 대조하여 복용 순응도 크로스 체크

#### 3. 약물-기분 상관관계 분석 (Visualization)

- **차트**: 기존 '감정 달력'에 '약물 복용 여부' 아이콘(💊)을 오버레이(Overlay)하여 표시
- **인사이트**: "약을 꾸준히 드신 날에는 기분 점수가 평균 15% 높았어요"와 같은 분석 멘트 제공

### 🚦 중증도 기반 맞춤형 케어 (Severity-Based Triage)

모든 사용자가 동일한 경험을 하는 것이 아니라, 증상의 경중에 따라 서비스 이용 흐름(User Flow)을 차별화합니다.

#### 1. 초기 선별 검사 (Screening)

- **시점**: 회원가입 직전 단계
- **도구**: 표준화된 자가 진단 도구 (예: PHQ-9 우울증 척도, GAD-7 불안 척도) 활용
- **분류**:
  - **경증 (Mild)**: 일반적인 감정 관리 및 예방 목적
  - **중등도 이상 (Moderate/Severe)**: 집중 관리가 필요한 위험군

#### 2. 중증도별 일기 작성 폼 분기 (Dynamic Diary Form)

- **경증 사용자**:
  - 자유 서술형 일기 위주
  - "오늘 하루 중 가장 기억에 남는 순간은?" 같은 가벼운 질문 제공
- **중증 사용자**:
  - 구조화된 질문지 위주 (행동 활성화 치료 기반)
  - "오늘 기분 점수(1-10)", "수면 시간", "식욕 변화" 등 생체 지표 필수 입력 유도
  - 챗봇이 "오늘 특별히 힘든 생각(자해 등)이 드셨나요?"와 같은 구체적 위험성 질문 수행

### 🚨 위기 대응 및 긴급 개입 (Crisis Intervention)

사용자의 생명을 보호하는 최후의 안전장치(Safety Net)를 구축합니다.

#### 1. 심층 분석 및 챗봇 인터뷰

- **트리거**: 일기 분석 결과 '매우 위험(심각한 우울, 자살 암시)' 판정 시
- **동작**:
  - 즉시 AI 상담 모드로 전환하여 추가 질문 수행 ("지금 혼자 계신가요?", "구체적인 계획이 있으신가요?")
  - 해당 대화 내용은 의료진 대시보드에 '긴급(Red Flag)'으로 즉시 전송

#### 2. 긴급 핫라인 연결 (Safety Plan)

- **상황**: 심야 시간대(센터 업무 종료)나 자살 임박 징후 감지 시
- **UI**: 화면 전체를 덮는(Full-screen) 긴급 도움 모달 팝업 띄움
- **연결**:
  - 📞 **자살예방상담전화 109** (24시간)
  - 📞 **정신건강위기상담전화 1577-0199**
  - 📞 **긴급구조 119**
- **기능**: 버튼 터치 한 번으로 즉시 전화 연결

## 3. 플랫폼별 적용 계획 (Implementation Plan)

| 구분            | 기능      | 웹 서비스 (B2B - 의료진용)                                | 앱 서비스 (B2C - 환자용)                                  | 구현 난이도 |
| :-------------- | :-------- | :-------------------------------------------------------- | :-------------------------------------------------------- | :---------- |
| **약물 관리**   | 정보 등록 | 환자 관리 페이지에서 처방 약물 확인 및 수정 기능          | '약물 알림 설정' 및 '복용 체크' UI 추가                   | 중 (Medium) |
|                 | 시각화    | 환자 상세페이지 '기분-약물 상관관계' 그래프 대시보드 추가 | 캘린더 화면에 '복용 완료' 스탬프 및 통계 제공             | 중 (Medium) |
| **중증도 관리** | 선별 검사 | 초기 문진 결과 조회 및 수동 등급 조정 기능                | 가입 시 PHQ-9 설문 조사 UI 및 결과에 따른 홈 화면 분기    | 하 (Low)    |
|                 | 작성 폼   | 환자 등급별로 '필수 수집 항목' 설정 기능 (커스텀 폼)      | 서버에서 내려받은 설정값에 따라 작성 폼 동적 렌더링       | 상 (High)   |
| **위기 대응**   | 위험 감지 | 대시보드 최상단 '고위험 환자' 실시간 알림 뱃지 표시       | 일기 분석 후 위험도 높으면 '긴급 대화' 시나리오 자동 시작 | 상 (High)   |
|                 | 긴급 연결 | 야간/주말 당직자용 알림 문자(SMS) 발송 시스템 구축        | 화면 하단 고정 'SOS 버튼' 및 위험 감지 시 팝업 노출       | 하 (Low)    |

## 4. 기대 효과

1.  **임상적 유효성 확보**: 약물 순응도와 기분 변화의 상관관계를 데이터로 입증하여 디지털 치료기기(DTx)로서의 가치 상승.
2.  **이탈률 감소**: 경증 환자에게는 가볍게, 중증 환자에게는 전문적으로 다가감으로써 앱 사용 부담감 완화.
3.  **환자 안전 강화**: 심각한 위기 상황에서 골든타임을 놓치지 않고 즉각적인 사회적 안전망(109, 119)으로 연결.
