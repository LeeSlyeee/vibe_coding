## P0: 즉시 착수 (이번 주 내)

### 수정 1: "한마디" — 로컬 LLM 전용 감정 표현 기능

**문제**: 우울증 환자는 "글 쓸 에너지가 없다"고 호소하지만, 이모지만 누르게 하면 감정 일기의 본질(감정의 언어적 표현)이 훼손됨.

**해결 방향**: 기존 "대화" 탭을 **"한마디"**로 리브랜딩. AI가 먼저 짧은 질문을 던지고, 사용자는 한두 문장만 답하는 2~3턴 초단문 대화. **로컬 LLM 전용**(서버 전송 0%)으로 On-Device AI의 킬러 피처로 포지셔닝.

**수정 내용**:

- 하단 탭: "대화" → **"한마디"** (아이콘 유지)
- 화면 진입 시 AI가 먼저 질문: "오늘 하루 중 가장 기억에 남는 순간이 있었나요?"
- 사용자 답변 → AI 1회 공감 + 후속 질문 → 사용자 마지막 답변 → 마무리 (총 2~3턴)
- 서버 AI 전환 옵션 제거 — 로컬 LLM 전용으로 고정
- 화면 상단에 "🔒 이 대화는 폰 안에서만 처리됩니다" 배지 상시 표시

**수정 파일**:

- `ios_app/AppMainTabView.swift` — 탭 이름 "대화" → "한마디"
- `ios_app/AppChatView.swift` — AI 선제 질문 로직 + 서버 AI 토글 제거 + 보안 배지
- `android_app/.../MainScreen.kt` — 탭 이름 변경
- `android_app/.../ChatScreen.kt` — 동일 적용
- `android_app/.../ChatViewModel.kt` — 서버 AI 모드 제거, 로컬 전용
- `frontend/src/views/ChatDiaryPage.vue` — 웹 동일 적용

### 수정 2: AI 위기 감지 한계 고지 문구

**문제**: 9명 중 5명(의사, 정치인, 대통령, 장관, 보호자)이 "AI가 위기를 놓치면 누가 책임지는가"를 지적. 현재 면책 고지는 '의료 행위 대체 아님'만 언급하고, **위기 감지의 기술적 한계**는 별도로 고지하지 않음.

**추가할 문구**:

> "마음온의 위기 감지 기능은 보조적 수단이며, 100% 정확성을 보장하지 않습니다. AI가 감지하지 못하는 위기 상황이 발생할 수 있으므로, 긴급한 상황에서는 즉시 자살예방상담전화 1393 또는 119에 직접 연락해 주세요."

**수정 파일**:

- `ios_app/AppSettingsView.swift` — 면책 고지 섹션에 위 문구 추가
- `android_app/.../SettingsScreen.kt` — 동일
- `frontend/src/views/SettingsPage.vue` — 동일
- `ios_app/AppChatView.swift` — 채팅 시작 시 한 번 표시
- `android_app/.../ChatScreen.kt` — 동일

### 수정 3: 보호자 행동 가이드(Playbook)

**문제**: 보호자가 "위기 알림을 받았는데 내가 뭘 해야 할지 모르겠다"고 호소. 현재 알림은 '위기 감지됨'만 전달하고 대응 방법 안내가 없음.

**수정 내용**:

- 보호자 알림 메시지에 **구체적인 3단계 행동 지침** 포함:
  1. "침착하게 전화 또는 문자로 안부를 확인하세요"
  2. "연락이 되지 않으면 가까운 경찰서(112)에 안전 확인을 요청하세요"
  3. "자살예방상담전화 1393에서 보호자 상담도 가능합니다"
- 보호자 설정 화면에 '위기 시 대응 가이드' 별도 섹션 추가

**수정 파일**:

- `backend/share_routes.py` — guardian-alerts 응답에 action_guide 필드 추가
- `ios_app/ShareManager.swift` — 알림 표시 시 가이드 포함
- `android_app/.../ShareScreen.kt` — 동일

### 수정 5: 앱스토어 브랜딩 톤다운

**문제**: 일반인이 "정신건강 앱이라는 카테고리 자체가 부담" — 낙인 효과(Stigma). 앱스토어 등록 정보에 '정신건강', '우울증' 같은 단어가 전면에 나오면 다운로드 저해.

**수정 방향**:

- '위기 감지', '자살 예방' 같은 키워드는 앱 설명 하단 기능 목록에만 간략 표기

**수정 파일**:

### 수정 6: 위기 텍스트 감지 시 '안전 폴백' 응답 전환

**문제**: 상담사와 환자 모두 "AI가 핀트 어긋난 답을 하면 오히려 2차 상처"를 우려. 2B 모델이 자유 생성한 위로 메시지가 맥락을 놓칠 위험이 있음.

**수정 내용**:

- 위기 Level 2 이상 감지 시, AI 자유 생성 응답을 **차단**하고, 사전 검증된 고정 응답 모듈(Safe Fallback)로 즉시 전환
- 고정 응답 예시:
  - "지금 많이 힘드신 것 같아요. 혼자 감당하지 않으셔도 됩니다."
  - "자살예방상담전화 1393은 24시간 운영됩니다. 전화해 보실 수 있을까요?"
- 현재 Android LLMService.kt에는 이미 `detectEmotion()` 함수에 위기 fallback이 있으나, iOS LLMService.swift와 서버 standalone_ai.py에도 동일 로직 적용 필요

**수정 파일**:

- `ios_app/LLMService.swift` — 위기 감지 시 로컬 LLM 응답 대신 고정 메시지 반환
- `backend/standalone_ai.py` — 서버 AI에서도 위기 키워드 감지 시 고정 응답
- `runpod_serverless_exaone/src/handler.py` — EXAONE에서도 동일 로직

## P3: 2주~1개월 (비즈니스/전략 영역)

### 수정 10: B2G 위기 알림 대응 매뉴얼(Playbook) 문서

보건소/정신건강복지센터에 납품 시 앱과 함께 제공할 **현장 담당자용 위기 대응 매뉴얼** MD/PDF 문서 제작. AI 위기 감지 → 현장 담당자 대응 → 후속 조치의 프로세스를 명문화하여, '시스템이 아닌 현장의 프로세스'에 책임을 분산시키는 구조.

### 수정 11: 팀 빌딩 로드맵

투자자, 정치인, 장관 3개 이해관계자가 공통으로 지적한 '1인 팀 리스크' 해소를 위한 구체적 채용/파트너십 계획. 예비창업패키지 수료 후 공동창업자 또는 핵심 인력(RA 전문가, BD 담당) 영입 타임라인 수립.

### 수정 12: 파일럿 프로젝트 계획서

도봉구 정신건강복지센터 또는 협력 가능한 의료기관 1곳에서 3~6개월 실증 데이터를 확보하기 위한 구체적 파일럿 계획서 작성. PHQ-9 등 표준 척도와 마음 온도 점수 간 상관관계 데이터 수집.
