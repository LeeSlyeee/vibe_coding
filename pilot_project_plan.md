# 마음온(Maum-On) 파일럿 프로젝트 계획서 (P3-수정12)

> **작성일**: 2026-03-02
> **대상 기관**: 도봉구 정신건강복지센터 (또는 협력 가능 기관 1곳)
> **기간**: 3개월 (연장 시 6개월)
> **근거**: 의사·투자자·장관 시뮬레이션에서 "임상 데이터 없이는 신뢰할 수 없다"는 공통 지적

---

## 1. 목적

1. 마음온 앱의 **감정 온도 점수**와 PHQ-9 등 **표준 심리 척도** 간 상관관계 데이터 확보
2. AI 위기 감지 시스템의 **민감도(Sensitivity) / 특이도(Specificity)** 측정
3. B2G 납품 시 제시할 **실증 데이터(Real-World Data)** 확보
4. 현장 운영 매뉴얼(Playbook)의 실효성 검증

---

## 2. 연구 설계

### 2.1 연구 유형

- **관찰 연구 (Observational Study)** — 비중재, 전향적 코호트
- IRB(기관생명윤리위원회) 승인 필수

### 2.2 대상 모집

| 항목          | 내용                                   |
| :------------ | :------------------------------------- |
| **대상**      | 정신건강복지센터 등록 내담자           |
| **모집 인원** | 30~50명                                |
| **연령**      | 만 15세 이상                           |
| **선정 기준** | 스마트폰 사용 가능, 앱 설치 동의       |
| **제외 기준** | 급성 정신병적 상태, 의사결정 능력 부재 |

### 2.3 동의 절차

1. 연구 안내문 배포 (목적, 절차, 개인정보 처리 방침)
2. **서면 동의서** 수집 (미성년자: 법정대리인 동의 추가)
3. 연구 참여 철회 시 데이터 즉시 삭제 보장

---

## 3. 측정 도구

| 도구                 | 측정 내용                          | 빈도            | 방법         |
| :------------------- | :--------------------------------- | :-------------- | :----------- |
| **마음온 감정 온도** | 일별 감정 점수 (1~5점 → 온도 변환) | 매일            | 앱 자동 기록 |
| **PHQ-9**            | 우울 증상 심각도                   | 격주 (2주 간격) | 앱 내 설문   |
| **GAD-7**            | 범불안장애 선별                    | 격주 (2주 간격) | 앱 내 설문   |
| **마음온 AI 리포트** | 주간 감정 분석                     | 주 1회          | 자동 생성    |
| **위기 감지 로그**   | AI 위기 키워드 감지 기록           | 실시간          | 서버 로그    |
| **사용자 만족도**    | SUS(System Usability Scale)        | 종료 시         | 앱 내 설문   |

## 4. 연구 일정

```
[사전 준비] (4주)
  ├── Week 1:  IRB 신청서 작성 및 제출
  ├── Week 2:  기관 MOU 체결
  ├── Week 3:  앱 내 설문 기능 개발 (PHQ-9, GAD-7)
  └── Week 4:  대상자 모집 공고

[파일럿 기간] (12주)
  ├── Week 1-2:   온보딩 — 앱 설치, 사용법 안내, 동의서 수집
  ├── Week 3-4:   적응 기간 — 매일 감정 기록 독려
  ├── Week 5-6:   중간 평가 — PHQ-9/GAD-7 2차 측정
  ├── Week 7-8:   안정 운영 — 위기 감지 정확도 관찰
  ├── Week 9-10:  중간 평가 — PHQ-9/GAD-7 3차 측정
  └── Week 11-12: 종료 평가 — 최종 PHQ-9/GAD-7 + SUS

[사후 분석] (4주)
  ├── Week 1-2:  데이터 정리 및 통계 분석
  └── Week 3-4:  결과 보고서 작성
```

## 5. 분석 계획

### 5.1 주요 분석

| 분석 항목                       | 방법                     | 기대 결과                  |
| :------------------------------ | :----------------------- | :------------------------- |
| 감정 온도 ↔ PHQ-9 상관          | 피어슨/스피어만 상관분석 | r ≥ 0.5 (유의미한 상관)    |
| 감정 온도 변화량 ↔ PHQ-9 변화량 | 반복측정 상관            | 변화 방향 일치율 80% 이상  |
| AI 위기 감지 정확도             | 민감도/특이도 분석       | 민감도 ≥ 80%, 특이도 ≥ 70% |
| 앱 사용 리텐션                  | 일별 DAU, 주별 WAU       | 12주 후 리텐션 ≥ 40%       |

### 5.2 부가 분석

| 분석 항목     | 방법                                    |
| :------------ | :-------------------------------------- |
| 사용자 만족도 | SUS 점수 (68점 이상 = 양호)             |
| 오탐/미감지율 | 위기 감지 로그 vs 담당자 실제 판단 비교 |
| 사용 패턴     | 주중 vs 주말, 시간대별 기록 빈도        |

## 8. 기대 성과물

1. **마음온 감정 온도 vs PHQ-9 상관관계 논문/보고서**
   - B2G 제안서에 첨부할 핵심 근거 자료
2. **AI 위기 감지 정확도 리포트**
   - 민감도/특이도 수치 → 투자 유치 시 활용
3. **현장 운영 매뉴얼(Playbook) 검증판**
   - 오탐/미감지 사례 반영한 v2.0
4. **사용자 경험 리포트**
   - UX 개선 포인트, 리텐션 데이터

## 10. 후속 계획

```
파일럿 성공 시:
  → 2~3개 기관으로 확대 (멀티 사이트)
  → 초기창업패키지 지원 시 핵심 근거 자료로 활용
  → 보건복지부 디지털 정신건강 사업 공모 참여
  → SCI급 학술지 투고 (예: JMIR Mental Health)

파일럿 부분 성공 시:
  → 감정 온도 알고리즘 재보정
  → UX 개선 후 2차 파일럿 (3개월)

파일럿 실패 시:
  → 실패 원인 분석 보고서 작성
  → 피봇 방향 검토 (웰니스 앱 → 디지털 일기 플랫폼)
```
