# 구현 계획: UI/UX 고도화 및 접근 제어 (RBAC)

## 구현 목표

본 계획은 기본적인 채팅 다이어리 기능을 넘어, 사용자 등급에 따른 정교한 기능 제어와 프리미엄급 UX를 제공하기 위한 고도화 작업을 다룹니다.

## 진행 상황

### 1단계: 사용자 등급 기반 접근 제어 (완료)

- [x] **백엔드**: `/api/user/me` 엔드포인트 구현 및 주요 API(통계, 약물 등)에 권한 체크 데코레이터 적용.
- [x] **프론트엔드**: 로그인 시 사용자 `risk_level` 정보를 `localStorage`에 캐싱.
- [x] **네비게이션 가드**: `App.vue`에서 메뉴 클릭 시 등급 확인 후 '접근 제한 모달' 표시 로직 구현.

### 2단계: 채팅 UX 전면 개편 (완료)

- [x] **임시 저장/복구**: `ChatDiaryPage.vue` 진입 시 중단된 대화가 있으면 복구 여부를 묻는 모달 구현.
- [x] **상호작용 개선**: `resetChat`, `goBack` 등 파괴적인 액션에 대해 네이티브 `confirm` 대신 커스텀 모달 적용.
- [x] **AI 응답 대기**: AI 응답 생성 중 사용자의 중복 조작을 막고 대기 심리를 완화하는 'Thinking' 모달 적용.

### 3단계: 코드 품질 및 스타일 안정화 (완료)

- [x] **Linting**: 불필요한 CSS 룰셋 제거 및 표준 속성(`appearance`) 추가로 브라우저 경고 해결.
- [x] **코드 정리**: `App.vue` 등의 중복 임포트 정리.

## 향후 계획 (Next Steps)

- **접근 제한 페이지 디자인**: 현재 모달로 처리된 접근 제한 안내를, 유료 플랜 안내 등이 포함된 전용 마케팅 페이지로/모달로 확장 고려.
- **다국어 지원**: 현재 하드코딩된 한글 텍스트들을 i18n 리소스로 분리 (추후 필요 시).
- **테마 확장**: 현재 Light Mode 중심의 UI를 다크 모드까지 확장 고려.
