## 1. Understanding Summary (이해도 요약)

- **제품:** 마음온(Maum-On) B2C SaaS 프리미엄 구독
- **핵심 기능명:** "마음 브릿지" — 대상별 커스텀 깊이의 감정 데이터 공유
- **대상 고객:** 정신건강에 관심 있는 개인 사용자 (B2C 직접 구독)
- **구독료:** 월 ₩5,500
- **추가 프리미엄 기능:** 없음. 마음 브릿지 단독.

### 무료 vs 유료 경계

| 무료 (현재 상태 유지)        | 유료 (₩5,500/월)               |
| :--------------------------- | :----------------------------- |
| 감정 일기 작성               | 무료 기능 전부 포함            |
| On-Device AI 분석 (7개 항목) | **+ 마음 브릿지**              |
| 개인 감정 기록 열람          | → 가족/보호자/상담사에게 공유  |
|                              | → 대상별 정보 깊이 직접 커스텀 |
|                              | → 상담사용 대시보드 리포트     |

### 절대 원칙 (Non-Negotiable)

| 원칙                   | 내용                                          |
| :--------------------- | :-------------------------------------------- |
| 🚫 AI 상담 금지        | AI는 분석 도구일 뿐, 상담/코칭 역할 절대 불가 |
| 🚫 외부 인력 의존 제로 | 상담사 마켓플레이스/관리 없음                 |
| 🔒 프라이버시          | 일기 원문은 어떤 경로로도 기기를 떠나지 않음  |
| 👤 사용자 통제         | 공유 대상·깊이 모든 결정은 사용자 본인        |
| 🏗️ 1인 팀 호환         | 100% 자체 통제 가능한 구조                    |

### 비목표 (Non-Goals)

- ❌ 상담사 마켓플레이스
- ❌ AI 코칭/상담 기능
- ❌ B2G/B2B는 MVP 범위 밖 (장기 확장 대상)
- ❌ 마음 브릿지 외 추가 프리미엄 기능

## 2. 가정 (Assumptions)

1. ₩5,500이 타겟 사용자에게 심리적 부담이 낮은 가격대이다
2. "마음 브릿지" 단독으로 유료 전환의 충분한 동기가 된다
3. 일기 원문이 아닌 AI 분석 결과만으로도 상담사/가족에게 유의미한 정보가 전달된다
4. 사용자가 가족/상담사에게 감정 데이터를 공유할 의향이 있다
5. Apple 소규모 사업자 프로그램(15% 수수료) 적용 가능 (연매출 $1M 미만)

## 4. 아키텍처 설계

### 4-1. 데이터 흐름

```
[사용자 기기 (On-Device)]
    │
    ├── 감정 일기 작성
    ├── AI 분석 (Gemma 2 2B, 7개 항목)
    ├── 분석 결과 로컬 저장
    │
    ├──── 가족/보호자 공유 ────────────┐
    │     기기에서 이미지/PDF 생성       │
    │     → 카카오톡/이메일 직접 전송    │
    │     (서버 미경유, 저장 없음)       │
    │                                   │
    └──── 상담사/의료진 공유 ──────────┐
          사용자가 허용한 항목만         │
          AES-256 암호화 후 서버 업로드  │
          → 상담사 전용 웹 대시보드      │
          (PIN 보호, 무기한 보관)        │
                                       ▼
                               [수신자가 열람]
```

### 4-2. 데이터 분류 및 보호 수준

| 데이터                  | 저장 위치                | 공유 가능 여부                                      |
| :---------------------- | :----------------------- | :-------------------------------------------------- |
| 일기 원문               | 📱 기기 전용             | 🚫 **절대 불가** — 어떤 경로로도 기기를 떠나지 않음 |
| AI 분석 결과 (7개 항목) | 📱 기기 전용             | ✅ 사용자가 허용한 항목만, 허용한 대상에게만        |
| 감정 점수/상태          | 📱 기기 전용             | ✅ 위와 동일                                        |
| 상담사 대시보드 데이터  | 🔐 서버 (AES-256 암호화) | ✅ 무기한 보관, 사용자 삭제 불가                    |
| 가족 공유 이미지        | ❌ 저장 안 함            | 기기에서 생성 → 직접 전송 → 서버에 흔적 없음        |

## 8. 프라이버시 & 보안

### 8-1. 상담사 대시보드 보안 플로우

```
[사용자가 "상담사 공유" 활성화]
    │
    ├── 1. 허용 항목만 추출
    ├── 2. AES-256 암호화
    ├── 3. 서버 업로드 (HTTPS/TLS 1.3)
    ├── 4. 고유 링크 + PIN 생성
    ├── 5. 사용자가 상담사에게 링크+PIN 직접 전달
    │
    [상담사 열람]
    ├── 6. PIN 입력 → 복호화 → 열람
    ├── 7. 열람 즉시 사용자에게 알림 푸시
    │
    [데이터 보관]
    └── 8. 무기한 보관 (상담 기록으로 취급)
```

### 8-2. 상담사 데이터 정책 (핵심)

> **상담사에게 공유된 데이터 = 상담 기록**

| 상태                     | 상담사 접근                                      | 데이터 보관             |
| :----------------------- | :----------------------------------------------- | :---------------------- |
| **공유 중**              | ✅ 과거 + 실시간 데이터 열람                     | 무기한 보관             |
| **사용자가 "접근 해제"** | ✅ 과거 데이터만 열람 가능 (새 데이터 전송 중단) | 무기한 보관             |
| **사용자 회원 탈퇴**     | ⏳ 1개월 유예                                    | 1개월 후 자동 영구 삭제 |

- 🚫 **사용자가 상담사의 데이터를 삭제할 수 없음** — 상담 기록의 법적·임상적 보존 의무
- 유일한 삭제 조건: **회원 탈퇴 후 1개월 경과 시 자동 삭제**

### 8-3. 사용자 통제 장치

| 기능           | 설명                                                |
| :------------- | :-------------------------------------------------- |
| **접근 해제**  | 새 데이터 전송 중단 (과거 기록은 상담사에게 유지)   |
| **열람 알림**  | 누가 언제 내 데이터를 봤는지 푸시 알림              |
| **공유 이력**  | 과거 공유 기록 전체 확인 가능                       |
| **기본값 OFF** | 모든 공유 항목이 기본 비활성, 사용자가 직접 켜야 함 |
