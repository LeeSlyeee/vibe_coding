## ✅ PASS 방향으로 작용하는 요소 (강점)

### 1. 작동하는 프로토타입 — "말만 하는 팀"과의 확실한 차별화
- iOS + Android + 백엔드 + 대시보드 + AI 엔진까지 **실제로 돌아가는 물건**을 보유
- 예비창업패키지에서 "아이디어만 있는 팀" vs "만들어 놓은 팀"의 체감 격차는 큼
- **심사위원 입장**: "이 사람은 진짜 만들 수 있구나" → 실행력 신뢰 확보

### 2. On-Device AI — 구조적으로 방어 가능한 기술 차별점
- "API 비용 제로"는 단순한 마케팅 문구가 아니라 **비즈니스 모델의 근간**
- 정신건강 데이터의 프라이버시 문제는 실제로 시장 진입 장벽 → 이것을 기술로 해결
- 심사위원 중 기술 관심자 1~2명은 이 포인트에서 확실히 끌림

### 3. B2G 진입 전략 — 예비창업패키지의 "사회적 가치" 평가 기준과 정렬
- 보건소, 정신건강복지센터 대상 → 공공성 어필 명확
- 자살 예방, 정신건강 사각지대 해소 → 정부 정책 방향과 일치
- 도봉구 정신건강복지센터와의 협력 계획 → MOU는 없지만 구체적인 접점 존재

### 4. 실비용 구조의 합리성
- 4,000만원(1단계 2,000만 + 2단계 2,000만) 규모의 소박한 예산
- AI/ML 엔지니어 채용, UX 외주, 파일럿 운영 등 **금액 대비 결과물이 명확**

## 🔴 FAIL 방향으로 작용하는 요소 (치명적 약점)

### 약점 1: 1인 팀의 구조적 리스크 — 심사위원의 **가장 큰 우려 포인트**

> [!CAUTION]
> 이것이 이 사업계획서의 **최대 약점**입니다.

**심사위원의 사고 흐름**:
```
"혼자서 iOS, Android, 백엔드, AI, 대시보드를 다 만들었다고?
→ 대단하긴 한데...
→ 이 사람이 아프면? 포기하면? 지치면?
→ 4,000만원 줬는데 사업이 중단되면?
→ 너무 리스크가 크다."
```

**전제조건 1("나보다 잘하는 사람이 많다")의 영향**:
- 비슷한 실력자들 중에는 **이미 2~3인 팀을 구성한 사람**이 분명히 있음
- 심사위원은 동일 점수대에서 **1인 vs 3인 팀** 중 선택해야 할 때, 3인 팀에 투표
- "채용 예정('26.08, '26.09)"은 **아직 없다는 뜻** → 신뢰 약함

**방어 전략**:
- "자문단(임상심리전문가, 정신건강의학과 전문의)"을 더 부각
- 가능하다면 공동창업자(CTO급) 1명이라도 합류 의향서를 확보
- **최소한** AI/ML 엔지니어 후보를 특정하여 "이 사람을 채용할 계획"이라고 명시

---

### 약점 2: "AI로 만들었다"가 2026년에는 차별점이 아님

> [!IMPORTANT]
> 전제조건 2("AI를 적극적으로 활용한다")를 적용하면, 이 사업계획서의 강점 상당수가 희석됩니다.

**현실 인식**:
- 2026년 기준, **AI 코딩 도구(Cursor, Claude Code, Copilot 등)를 활용한 1인 개발자가 6개 플랫폼을 구축하는 것은 더 이상 초인적인 능력이 아님**
- "풀스택 1인 개발"의 감탄 포인트가 2025년에 비해 현저히 감소
- 경쟁 신청자 중에서도 AI로 프로토타입을 빠르게 만들고 지원하는 팀이 다수 존재할 것

**심사위원 중 AI 트렌드에 밝은 사람의 시선**:
```
"AI로 빠르게 프로토타입을 만든 건 좋은데...
→ 이게 정말 깊이 있는 기술인가, 아니면 표면적 구현인가?
→ On-Device AI라고 했는데, Gemma 2B 4-bit로 정신건강 상담 품질이 진짜 나오나?
→ GPT-4급 모델의 상담 품질 vs 2B 모델의 상담 품질... 차이가 너무 클 텐데?"
```

**이 약점의 핵심**: On-Device AI의 **"실제 상담 품질"에 대한 증거가 없음**
- 논문도 없고, 임상 데이터도 없고, 사용자 만족도 조사도 없음
- "실시간 추론 성공"은 **속도** 이야기이지 **품질** 이야기가 아님
- 2B 모델이 정신건강 영역에서 유의미한 결과를 낸다는 **증거(Proof)**가 부재

### 약점 3: B2G 진입 전략의 실현 가능성 의문

**심사위원의 의구심**:
```
"보건소·정신건강복지센터에 납품하겠다고?
→ 실적(납품 이력)이 있나? → 없음
→ MOU라도 있나? → '협력 시기: 2026.09' (=아직 없음)
→ 보건복지부 시범사업 공모에 어떻게 참여하지? 법인도 설립 전인데?
→ 보건소가 인증도 없는 AI 앱을 도입할 이유가 있나?"
```

**구체적 허점**:
- 도봉구 정신건강복지센터와의 협력은 **계획일 뿐 확정이 아님**
- "보건소 코드 기반 환자-상담사 매칭"을 구현했다고 하지만, 실제 보건소와 연동 테스트를 한 적은 없음
- DTx(디지털 치료제) 인증은 **최소 2~3년**, **수억 원**이 필요한 과정 — 4,000만원으로 "착수"라고 쓰는 것 자체가 비현실적

---

### 약점 4: 재무 계획의 빈약함

**수익 예측이 사실상 없음**:
- "기관 라이선스 월 50~100만원" → 몇 개 기관에 언제 납품하여 매출이 얼마인지?
- "정부 시범사업 건당 5~20억" → 이것은 희망사항이지 예측이 아님
- **Break-even point(손익분기점)**이 제시되지 않음
- 심사위원의 경영/재무 전문가는 이 부분에서 확실히 감점

**비용 구조의 의문점**:
- AI/ML 엔지니어 3개월 × 300만원 = 900만원 → **이 연봉(세전 3,600만원)으로 AI/ML 석사 이상 인력을 채용할 수 있는가?**
- GPU 클라우드 3개월 × 100만원 → 파인튜닝에 충분한가?

---

### 약점 5: "의료 행위"와의 경계선 모호

**법적/규제 리스크**:
- AI가 "상담"을 제공한다고 표현 → **의료법상 무면허 의료행위**에 해당할 가능성
- "위기 감지(Red Flag)"로 자살·자해를 탐지한다고 했는데, **오탐(False Positive)이나 미탐(False Negative) 시 법적 책임은?**
- 이 부분에 대한 법률 자문 결과나 방어 논리가 사업계획서에 부재

## 🎯 경쟁 구도에서의 포지셔닝 분석

### 예비창업패키지 경쟁 구도 시뮬레이션

```
[일반적인 신청자 풀]

Tier A (상위 10%): 
  - 3~5인 팀 + 이미 매출 발생 + VC 관심 + 공동창업자에 업계 경력자
  → 이 사람들과 직접 경쟁하면 패배할 가능성이 높음

Tier B (상위 10~30%): ← 현재 마음온의 추정 포지션
  - 1~2인 팀 + 작동하는 프로토타입 + 기술적 차별점 보유
  - 팀 구성이 약하거나, 시장 검증이 부족하거나, 매출이 없음
  → 이 구간에서의 경쟁이 관건

Tier C (하위 70%):
  - 아이디어만 있거나, PPT만 있거나, 기술이 너무 범용적
  → 마음온은 이 그룹보다 분명히 우위
```

### 마음온이 Tier B에서 상위로 올라가기 위해 필요한 것

1. **팀**: 최소 1명의 공동 창업자/확정 멤버 (=의향서라도)
2. **검증**: 사용자 10명이라도 beta 테스트한 결과 데이터
3. **MOU**: 보건소/상담센터 1곳이라도 확정된 파일럿 약속

## 🔬 전제조건별 심층 분석

### 전제조건 1: "나보다 잘하는 사람이 많다"

**영향**: 기술적 우수성만으로는 합격이 보장되지 않음

- 예비창업패키지는 **기술 심사가 아님** — 사업성, 팀, 시장성, 실현 가능성을 종합 평가
- "기술은 좋은데 팀이 약하다"는 흔한 탈락 사유
- 비슷한 수준의 기술력을 가진 3인 팀이 헬스케어로 지원한다면, 해당 팀에 점수가 밀릴 가능성 높음

### 전제조건 2: "AI를 적극적으로 활용한다"

**영향**: "1인이 이 모든 걸 만들었다"의 임팩트 감소

- 2024년이었다면 "AI로 혼자 이걸 다 만들었다고?!" → 감탄 + 가산점
- 2026년에는 "어, 요즘은 AI 쓰면 그 정도는 되지" → 평범한 수준으로 인식
- **차별화 포인트가 "만들었다는 사실"에서 "만든 것의 깊이와 품질"로 이동**
- 따라서 On-Device AI의 **상담 품질, 임상 효과, 사용자 만족도** 같은 **정성적 증거**가 훨씬 더 중요해짐

## 💡 합격 확률을 60~70%로 올리기 위한 급선무 (우선순위 순)

> [!TIP]
> 아래 3가지 중 **1번은 반드시**, 2~3번 중 하나를 추가로 확보하면 확률이 유의미하게 상승합니다.

### 1. 🔴 [필수] 팀원 확보 또는 증거
- "함께 할 AI/ML 엔지니어 후보"의 이력서 또는 합류 의향서
- 혹은 "보조 개발자 1명"이라도 이름을 넣는 것
- **이것 하나가 합격 확률을 10%p 이상 올림**

### 2. 🟡 [강력 권장] 베타 테스트 결과 데이터
- TestFlight/Play Console 베타 사용자 10~20명의 사용 데이터
- "감정 분석 정확도 82%", "사용자 만족도 4.2/5.0" 같은 수치
- 정량적 데이터 1줄이 정성적 설명 1페이지보다 강력함

### 3. 🟡 [권장] 보건소/상담센터와의 접촉 증거
- 도봉구 정신건강복지센터와의 미팅 기록, 이메일, 관심 표명 문서
- "2026.09 예정"이 아니라 "이미 논의 중"이라는 증거
