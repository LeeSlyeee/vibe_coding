## [슬라이드 2] 문제 인식 — 정신건강의 3대 사각지대

### 우리 사회의 마음 건강은 위기입니다

**1. 의료 접근의 벽**

- 전문의 1인당 담당 환자 수 초과 — 내원 사이 관리 공백
- 진료실 밖은 '블랙박스' — 환자의 일상 감정 변화 파악 불가
- 재발 징후 조기 포착 실패

**2. 기록의 저항감**

- "숙제"에 대한 부담 → 기록 포기 → 치료 효과 저하
- 기억 왜곡으로 인한 부정확한 진술
- 수기 기록의 분석 한계

**3. 데이터 보안 장벽**

- 민감한 감정 데이터를 외부 클라우드에 올리는 것에 대한 불안
- 개인정보보호법 + 의료법 규제 — 기존 AI 챗봇 도입 장벽
- 중소 병원·보건소는 자체 서버 구축 비용 감당 불가

> 핵심 질문: "안전하면서도, 누구나 부담 없이 사용할 수 있는 디지털 멘탈케어는 없을까?"

## [슬라이드 3] 솔루션 — 마음온이란?

### 가장 안전한 방식으로 마음의 온도를 지켜주는 AI

마음온(Maum-On)은 On-Device AI 기술을 기반으로 한 **감정 기록 보조 도구**입니다.

**핵심 가치 3가지:**

1. **Network-OFF, AI-ON**
   - 인터넷 연결 없이도 AI가 폰 내부에서 작동
   - 감정 데이터가 외부 서버로 전송되지 않음 (데이터 전송 0%)
   - 비행기 모드에서도 상담 가능

2. **마음의 온도를 측정하는 케어**
   - 매일 기록되는 감정을 0~100° 마음 온도로 시각화
   - AI가 따뜻한 공감과 인정을 제공 (공허한 격려 대신 진정한 위로)
   - 감정 패턴을 장기 추적하여 변화 통찰 제공

3. **의료진과의 안전한 연결다리**
   - 환자 동의 하에만 의료진에게 마음 온도 + 위험 신호 공유
   - 대화 내용은 절대 외부 비공개
   - 위기 상황 시 자동 알림으로 골든타임 확보

> 마음온은 의료 행위를 대체하지 않습니다. 전문 의료 서비스의 보조 도구로서 기능합니다.

## [슬라이드 4] 제품 구성 — 3대 플랫폼

### 환자용 앱 + 의료진 대시보드 + 관리 시스템

**1. 환자용 모바일 앱 (iOS / Android)**

- 감정 일기 기록 — AI가 선제적으로 "오늘 하루 어떠셨나요?" 질문
- On-Device AI 감정 분석 — Gemma-2-2b 엔진으로 로컬 추론
- 마음 온도 시각화 — 일별/주별/월별 감정 추이 차트
- AI 공감 대화 — 위로와 경청, 절대 진단이나 처방이 아닌 공감
- 3단계 위기 감지 시스템 — 자살예방상담전화 1393 원터치 연결

**2. 웹 프론트엔드 (Vue.js)**

- 동일한 감정 일기 + AI 분석 + 마음 온도 기능
- PC 환경에서의 편리한 기록 지원
- 반응형 디자인으로 태블릿 호환

**3. 의료진 대시보드 (InsightMind)**

- 환자 감정 추이 실시간 모니터링
- Red Flag(위험 신호) 알림 — 위기 감지 시 즉시 알림
- 데이터 기반의 정밀한 진료 보조 자료 제공
- 대화 원문 미공개 — 마음 온도 + 감정 점수만 전달

## [슬라이드 5] 핵심 기술 — On-Device AI

### 병원 안으로 들어온 안전한 AI

**On-Device AI란?**

- AI 추론이 클라우드가 아닌 사용자의 스마트폰 내부에서 수행
- 네트워크 연결 없이도 완전한 기능 작동
- 감정 데이터의 물리적 외부 유출 원천 차단

**탑재 엔진: Google Gemma 2 (2b)**

- Apple Silicon(MLX) 및 MediaPipe 기반 최적화 완료
- iPhone 15 Pro 이상: 즉시 로컬 분석 (로딩 2~3초)
- 저사양 기기: 서버 AI 자동 전환 (암호화 전송 + 즉시 파기)

**하이브리드 AI 아키텍처**

- 1순위: On-Device AI (폰 내 Gemma 2) — 프라이버시 최고
- 2순위: 자사 서버 AI (EXAONE / Gemma) — 암호화 통신
- Fallback: 로컬 Ollama 서버 — 자동 전환, 사용자 무감지

**보안 설계:**

- 비행기 모드에서도 작동
- 데이터 전송 0%
- 개인정보보호법 및 의료법 완벽 대응
- HIPAA 가이드라인 준수 설계

## [슬라이드 6] 안전 설계 — 위기 대응 프로토콜

### 생명을 지키는 3단계 위기 감지 시스템

마음온은 사용자의 안전을 최우선으로 설계했습니다.

**Level 1 — 관심 (Concern)**

- 감지 조건: "힘들어", "우울해", "불안해" 등 감정 표현
- 대응: AI 공감 모드 전환 → 경청과 위로 강화
- 플랫폼: iOS, Android, Web 전체 적용

**Level 2 — 주의 (Warning)**

- 감지 조건: "사라지고 싶어", "살기 싫어", "의미 없어" 등
- 대응: 위기 배너 노출 + 전문 상담 정보 안내
- 의료진/보호자에게 알림 전달 (동의 시)

**Level 3 — 위험 (Emergency)**

- 감지 조건: "자살", "죽고 싶어", "유서", "목을" 등 즉각적 위험 표현
- 대응: SOS 모달 즉시 표시 + 원터치 긴급 연결
  - 자살예방 상담전화 1393
  - 정신건강 상담전화 1577-0199
  - 경찰청 긴급신고 112
- 전체 보호자·의료진 동시 알림 (보호자 알림 모드 활성화 시)

**AI 금지 표현 정책:**

- "힘내세요", "긍정적으로 생각하세요", "잘 될 거예요" 등 공허한 격려 사용 금지
- 시스템 프롬프트에 명시 + AI 응답 후처리 필터로 이중 방어
- 대신 공감, 인정, 구체적 질문으로 대응

## [슬라이드 7] 보호자 알림 모드

### 환자 동의 하의 투명한 돌봄 네트워크

**보호자 등록 시스템:**

- 환자 본인이 동의한 경우에만 보호자 1~2명 등록 가능
- 보호자 역할(guardian) vs 일반 조회자(viewer) 분리

**보호자에게 전달되는 정보 (3종):**

1. 마음 온도 급락 알림 — 최근 평균 기분이 2.0 이하로 떨어졌을 때
2. 위기 신호 알림 — safety_flag 감지 시 즉시 알림
3. 장기 미기록 알림 — 7일 이상 기록이 없을 때 알려줌

**환자 주도 공유 모델 (Phase 6):**

- 환자가 직접 "어떤 정보를 공유할지" 선택 가능
  - 마음 온도만 공유 / 위기 신호만 공유 / 주간 리포트 공유
- 대화 내용은 **절대** 보호자에게 비공개
- 투명성: "보호자에게 어떤 데이터가 전달되는지" 설정 화면에서 확인 가능

**API 구현 완료:**

- PUT /api/share/role — 보호자 역할 설정
- PUT /api/share/alert-settings — 알림 설정
- PUT /api/share/share-scope — 공유 범위 제어
- GET /api/share/guardian-alerts — 알림 목록 조회
- GET /api/share/guardians — 보호자 목록 확인

## [슬라이드 8] 마음 온도 측정 체계

### 0°~100° 직관적 감정 시각화

사용자의 감정 상태를 숫자와 색상으로 즉시 이해할 수 있습니다.

**마음 온도 등급:**

- 🟢 80~100° — 안정: "오늘 하루 따뜻했네요"
- 🟡 60~79° — 흐림: "조금 흐린 하루였군요"
- 🟠 40~59° — 주의: "마음이 조금 무거우셨겠어요"
- 🔴 0~39° — 위험: 위기 대응 프로토콜 자동 발동

**산정 기준:**

- 감정 키워드 긍·부정 비율 (40%)
- 기록 빈도 (20%)
- 수면 시간 (20%)
- 활동량 (20%)

**시각화 기능:**

- 일별 마음 온도 캘린더 (이모지 표시)
- 주별/월별 추세 그래프
- AI 기반 감정 패턴 분석 리포트
- 의료진 대시보드에 자동 동기화

> "오늘 당신의 마음 온도는 몇 도인가요?"

## [슬라이드 9] 서비스 포지셔닝 — 의료 행위가 아닌 감정 기록 보조 도구

### "감정 케어 도우미"로서의 명확한 위치

**마음온은 다음을 하지 않습니다:**

- ❌ 진단을 내리지 않습니다
- ❌ 처방을 제시하지 않습니다
- ❌ 의료 행위를 대체하지 않습니다
- ❌ "힘내세요"와 같은 공허한 격려를 하지 않습니다

**마음온이 하는 것:**

- ✅ 매일의 감정을 기록하고 정리합니다
- ✅ AI가 따뜻하게 공감하고 경청합니다
- ✅ 감정 패턴을 시각화하여 자기 인식을 돕습니다
- ✅ 위기 상황에서 전문 상담 기관을 즉시 연결합니다
- ✅ 의료진에게 객관적인 데이터를 보조 자료로 제공합니다

**면책 고지 — 전 플랫폼 일관 적용:**

- iOS: 설정 화면, 일기 분석 화면, 채팅 인트로 (3곳)
- Android: 설정 화면, 통계 화면, 일기 상세, 채팅 인트로 (4곳)
- Web: 설정 페이지, 리포트 화면, 일기 모달 (3곳)

> "본 서비스는 의료 행위, 심리 치료, 또는 전문 상담을 대체하지 않습니다.
> AI가 제공하는 분석과 코멘트는 참고용이며, 의료적 판단으로 간주될 수 없습니다."

## [슬라이드 10] 경쟁사 비교 분석

### 마음온 vs 기존 멘탈케어 앱 (Mindy, 마인드카페, 트로스트)

| 비교 항목    | 기존 앱 (Mindy 등)        | 마음온                          |
| :----------- | :------------------------ | :------------------------------ |
| 타겟 시장    | B2C (일반 대중)           | B2B / B2G (의료기관, 보건소)    |
| 데이터 보안  | 클라우드 중앙 서버        | On-Device AI + 로컬 저장        |
| AI 처리 위치 | 서버 기반 (인터넷 필수)   | 디바이스 내부 (오프라인 가능)   |
| 의료진 연계  | 없음 (단순 개인 기록)     | 의료진 대시보드 + Red Flag 알림 |
| 보호자 알림  | 없음                      | 3종 알림 + 환자 주도 공유 제어  |
| 위기 대응    | 단순 안내 문구            | 3단계 분류 + 1393 원터치 연결   |
| 규제 대응    | 웰니스 앱 (비의료기기)    | DTx 인허가 추진 가능            |
| 비용 구조    | 개인 구독료 (이탈률 높음) | 기관 라이선스 + 공공 조달       |
| 프라이버시   | 약관 동의 기반 수집       | 물리적 유출 불가 (On-Device)    |
| 고령층 접근  | 높은 스마트폰 숙련도 필요 | 음성 인식 + AI 선제 질문        |

**마음온만의 압도적 우위:**

1. 데이터 전송 0% — 개인정보보호법 완벽 준수
2. 의료진 워크플로우 통합 — 진료 효율 극대화
3. B2B/B2G 장기 계약 — 안정적 수익 구조
4. 블루오션 선점 — 의료기관용 AI 멘탈케어 시장은 경쟁자 전무

## [슬라이드 11] 비즈니스 모델

### 안정적 수익 구조 — B2B/B2G 복합 모델

**1. SaaS 구독 모델 (민간 병원 및 심리상담센터)**

- 타겟: 정신건강의학과, 심리상담센터
- 수익 구조: 의사 1인당 월 15~30만원 구독료 (Seat-based)
- 기대 규모: 정신과의사 4,500명 × 연 180만원 ≈ 연 81억원

**2. 공공 조달 모델 (보건소 및 정신건강복지센터)**

- 타겟: 전국 268개 보건소, 정신건강복지센터, 치매안심센터
- 수익 구조: 센터당 연 500~1,000만원 라이선스
- 기대 규모: 268개 센터 × 750만원 ≈ 연 20억원

**3. 지자체 패키지 (B2G 사업)**

- 타겟: 지자체 '고독사 예방', '청년 마음건강' 사업 예산 활용
- 수익 구조: 사업당 3,000~5,000만원 일괄 납품 + 유지보수
- 기대 규모: 연 50건 × 4,000만원 ≈ 연 20억원

**시장 규모 (TAM/SAM/SOM):**

- TAM (전체 시장): 국내 디지털 멘탈케어 시장 약 3,000억원
- SAM (접근 가능): B2B/B2G 의료기관용 AI 도구 약 500억원
- SOM (초기 목표): 정신과 + 보건소 시장 선점 약 50억원
