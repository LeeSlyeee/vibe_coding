## 1. 🩺 정신과의사의 부정적 의견

### 1-1. "AI 상담"이라는 표현이 의료법 위반 소지

| :-------------- | :-------------------- |
| **해결 가능성** | 🟢 즉시 해결 가능     |

### 1-2. 임상적 유효성 근거 부재 (KPI 수치의 출처 없음)

| 항목            | 내용                |
| :-------------- | :------------------ |
| **해결 가능성** | 🟡 조건부 해결 가능 |
| **난이도**      | 높음 (파일럿 필요)  |
| **소요 시간**   | 3~6개월             |

**조치 방안:**

- **즉시(1주)**: 소개서의 "참여율 3배 향상, 진료 시간 50% 단축"을 **"목표 KPI"로 명시**하거나, 출처가 없는 수치는 제거. "검증 예정"이라고 정직하게 표기
- **단기(1~3개월)**: 1개 병원 또는 보건소와 **파일럿 MOU 체결**. 소규모(환자 20~50명)라도 실제 데이터 확보
- **중기(3~6개월)**: 파일럿 결과를 논문 또는 백서로 발표. 사전-사후 비교(Before-After) 디자인으로 유효성 근거 생성

### 1-3. 2B 경량 모델의 감정 분석 정확도 의문

| 항목            | 내용                  |
| :-------------- | :-------------------- |
| **해결 가능성** | 🟡 조건부 해결 가능   |
| **난이도**      | 중간~높음 (기술 개선) |
| **소요 시간**   | 2~4개월               |

**조치 방안:**

- **AI의 역할 재정의**: AI가 "감정을 진단"하는 것이 아니라, **"감정 키워드를 추출하고 빈도·패턴을 시각화"**하는 것으로 포지셔닝 변경. 의학적 판단은 의사의 몫
- **위험 신호 임계값(Threshold)을 의사가 설정**할 수 있도록 대시보드 커스터마이징 기능 추가
- **오탐/미탐 비율 측정**: 파일럿에서 AI 분석 결과 vs 의사의 실제 진단을 비교하여 정확도 데이터 축적
- **모델 업그레이드 로드맵**: Gemma 4B → 향후 더 큰 모델(4B, 7B)로 점진적 업그레이드 계획 수립. 디바이스 성능 향상에 따라 가능

**조치 방안:**

| **소요 시간**   | 지속적                    |

**조치 방안:**

- **비즈니스 모델에 "상담사 역량 강화" 형태를 추가**: AI 도구 비용이 상담사 채용 예산과 **별도 항목**에서 편성되도록 B2G 제안서를 설계

- **상담사 대상 교육 프로그램**을 별도 제공하여 도구 활용 역량을 높이는 방향 제시

### 2-3. 언어와 감정의 괴리 (한국어 "괜찮아요"의 다중 의미)

| 항목            | 내용                  |
| :-------------- | :-------------------- |
| **해결 가능성** | 🟡 조건부 해결 가능   |
| **난이도**      | 높음 (AI/NLP 연구)    |
| **소요 시간**   | 3~6개월 (지속적 개선) |

**조치 방안:**

- **다중 신호 분석**: 텍스트 내용만이 아닌, **응답 빈도, 응답 길이, 응답 시간대, 사용 패턴 변화** 등 행동 지표(Behavioral Signal)를 종합 분석
- 예: "괜찮아요"라고 했지만, 새벽 3시에 기록하고, 최근 7일간 기록 빈도가 급감했으면 → 행동 패턴상 위험 요소로 플래그
- **한국어 감정 분석 파인튜닝**: 한국어 감정 표현의 뉘앙스(예: "괜찮아요"의 진짜 의미)를 학습한 파인튜닝 데이터셋 구축
- **AI의 한계를 투명하게 고지**: "이 분석은 참고용이며, 최종 판단은 전문가가 합니다" 표기

### 2-4. 자살 충동 고백 시 AI의 윤리적 대응

| 항목            | 내용                        |
| :-------------- | :-------------------------- |
| **해결 가능성** | 🟢 즉시 해결 가능           |
| **난이도**      | 중간 (프로토콜 설계 + 구현) |
| **소요 시간**   | 2~4주                       |

**조치 방안:**

- **위기 대응 프로토콜 3단계 설계:**

```
[Level 1] 위험 키워드 감지 (죽고 싶다, 끝내고 싶다, 할 이유가 없다 등)
  → AI 대화 모드 즉시 전환: 공감 + 안전 확인 질문
  → "지금 많이 힘드시군요. 혹시 지금 안전한 곳에 계신가요?"

[Level 2] 자살 계획 구체화 감지 (방법, 시기, 장소 언급)
  → 자살예방상담전화(1393) 즉시 안내 + 원터치 전화 연결 버튼
  → 담당 의료진에게 긴급 알림 발송
  → 보호자(설정 시)에게 긴급 알림 발송

[Level 3] 즉각적 위험 판단
  → 119 응급 연결 안내
  → 모든 지정 보호자·의료진에게 동시 알림
  → AI 대화 종료, 위기 개입 화면으로 전환
```

- 이 프로토콜을 **한국자살예방센터 가이드라인**에 맞게 검증
- 소개서에 위기 대응 프로토콜 존재를 **명시적으로 기재**

## 3. 👤 일반인의 부정적 의견

### 3-1. B2C 채널 부재 (일반인이 직접 사용 불가)

| 항목            | 내용                  |
| :-------------- | :-------------------- |
| **해결 가능성** | 🟡 조건부 해결 가능   |
| **난이도**      | 중간 (사업 모델 확장) |
| **소요 시간**   | 3~6개월               |

**조치 방안:**

- **1차**: B2B/B2G 시장에서 레퍼런스를 확보한 후, 일반인용 **무료 라이트(Lite) 버전**을 앱스토어에 배포
  - Lite: 감정 기록 + 기본 AI 대화 (의료진 연동 없음)
  - Full: 의료진 대시보드 연동 (B2B/B2G 전용)
- **2차**: 보건소 이용자가 아니더라도 **지역 정신건강복지센터 코드**를 입력하면 사용 가능한 개방형 모델 검토
- 소개서에 B2C 로드맵을 **향후 계획**으로 기재

### 3-2. ChatGPT 대비 AI 대화 품질 우려

| 항목            | 내용                  |
| :-------------- | :-------------------- |
| **해결 가능성** | 🟡 조건부 해결 가능   |
| **난이도**      | 높음 (지속적 AI 개선) |
| **소요 시간**   | 지속적                |

**조치 방안:**

- **기대치 재설정**: ChatGPT와 비교 대상이 되지 않도록, **"자유 대화 AI"가 아닌 "구조화된 감정 체크인"**으로 포지셔닝
  - 매일 3~5개 질문을 통한 감정 스캐닝 → 자유 대화보다는 **가이드된 대화(Guided Conversation)** 방식
- **프롬프트 엔지니어링 고도화**: Gemma 4B의 응답 품질을 높이기 위한 시스템 프롬프트 최적화
- **멀티모달 보완**: 텍스트 대화 품질이 제한적이라면, **감정 이모지 선택, 슬라이더, 음성 톤 분석** 등 비텍스트 입력을 강화하여 대화 의존도를 낮춤
- **모델 로드맵**: 디바이스 성능 향상에 따라 Gemma 4B, 7B 순차 도입 계획 명시

### 3-3. 멘탈케어 앱의 지속 사용 문제 (2주 후 이탈)

| 항목            | 내용                     |
| :-------------- | :----------------------- |
| **해결 가능성** | 🟡 조건부 해결 가능      |
| **난이도**      | 중간 (UX/게이미피케이션) |
| **소요 시간**   | 2~3개월                  |

**조치 방안:**

- **AI 선제 알림(Proactive Push)**: 사용자가 앱을 열지 않아도 AI가 먼저 "오늘 어떠세요?" 알림을 보내는 구조. 기록하는 행위를 사용자가 시작할 필요 없음
- **마이크로 인터랙션**: 기록에 30초만 소요. "오늘 기분 3점/5점 중 골라주세요" 수준의 초간단 입력
- **시각적 보상**: 기록이 쌓이면 "마음 온도 그래프"가 완성되는 시각적 성취감
- **의료진 연동이 동기부여**: B2B/B2G 사용자는 "의사 선생님이 보시니까 기록해야지"라는 외부 동기(External Motivation)가 작용하여 B2C보다 이탈률이 낮을 것으로 예상

### 3-4. "마음의 온도" 컨셉의 모호함

| 항목            | 내용              |
| :-------------- | :---------------- |
| **해결 가능성** | 🟢 즉시 해결 가능 |
| **난이도**      | 낮음 (UX 정의)    |
| **소요 시간**   | 1~2주             |

**조치 방안:**

- **마음 온도 지표 구체화**: 0~100°까지의 점수 체계 정의
  - 예: 80~100° = 안정 / 60~79° = 흐림 / 40~59° = 주의 / 0~39° = 위험
- **구성 요소 명시**: 마음 온도 = (감정 키워드 긍·부정 비율 × 40%) + (기록 빈도 × 20%) + (수면 시간 × 20%) + (활동량 × 20%)
- 소개서와 앱 UI에 이 산정 기준을 사용자가 볼 수 있게 투명하게 표시

## 4. 😔 우울증 환자의 부정적 의견

### 4-1. AI 공감 능력의 한계 → 고립감 강화 위험

| 항목            | 내용                |
| :-------------- | :------------------ |
| **해결 가능성** | 🟡 조건부 해결 가능 |
| **난이도**      | 높음 (AI 응답 품질) |
| **소요 시간**   | 지속적 개선         |

**조치 방안:**

- **응답 톤 파인튜닝**: 임상심리 전문가와 협력하여 **따뜻하고 비판단적인(Non-judgmental) 응답 데이터셋** 구축. "그렇군요"의 반복이 아닌, 맥락에 맞는 다양한 공감 표현 학습
- **AI의 한계를 솔직하게 인정하는 응답 설계**: "저는 AI라서 완벽하게 이해하긴 어렵지만, 말씀해주신 것만으로도 의미 있어요." — 겸손한 AI가 거만한 AI보다 환자에게 덜 해롭다
- **인간 연결 유도**: AI 대화 중 일정 조건이 충족되면, "전문 상담사와 이야기해보시겠어요?"라는 **자연스러운 전문가 연결 제안**
- **침묵에도 대응**: 앱을 열었지만 아무 말도 하지 않는 경우에 대한 응답 설계. "말하고 싶지 않은 날도 있죠. 여기 있을게요."

### 4-2. 감시당하는 느낌 (의료진 모니터링에 대한 거부감)

| 항목            | 내용              |
| :-------------- | :---------------- |
| **해결 가능성** | 🟢 즉시 해결 가능 |
| **난이도**      | 중간 (기능 설계)  |
| **소요 시간**   | 2~4주             |

**조치 방안:**

- **환자 주도 공유 모델(Patient-controlled Sharing)**:
  - 전체 기록이 아닌, **요약 리포트만** 의료진에게 전달
  - 환자가 **어떤 정보를 공유할지 직접 선택**할 수 있는 옵션 제공
  - "이 기록은 담당 선생님에게 공유하시겠어요?" 확인 절차
- **익명화된 감정 점수만 공유**: 대화 원문은 절대 전달하지 않고, **수치화된 감정 점수(마음 온도)**와 **위험 신호 여부**만 전달
- 앱 내에 **"의료진에게 어떤 정보가 전달되는지"를 투명하게 안내하는 페이지** 상설
