# 마음온(Maum-On) 킥(Kick) 기능 계획서

> **작성일**: 2026-03-03
> **목적**: 마음온의 핵심 차별화 기능(킥)으로 선정된 기능들의 기획 의도, 체크리스트 검증 결과, 이해관계자 가치를 정리한다.
> **전제 조건**: 모든 킥은 `PRODUCT_PHILOSOPHY_CORE.md`의 가치 사슬 체크리스트를 통과해야 한다.

## 공통 원칙

마음온의 가치 사슬:

```
사용자가 자신의 생각을 글로 쓴다 (일기)
           ↓
On-Device AI가 텍스트를 분석한다
           ↓
마음 온도가 산출된다
           ↓
의료진 대시보드에 인사이트 표시
           ↓
위기 감지 → 알림 → 개입
```

> **모든 킥은 이 사슬의 원점(텍스트 입력)을 건드리지 않고, 기존 사슬 위에 새로운 분석 능력을 "얹는" 것이어야 한다.**

## 킥 #1: 시계열 패턴 감지기 (Time-Series Pattern Detector)

### 한 줄 정의

> **"사용자가 아무것도 안 해도, 침묵 자체를 신호로 읽는다."**

### 해결하는 문제

기존 시스템의 모든 기능(키워드 감지, SOS, 보호자 알림)은 **"사용자가 무언가를 해야"** 작동한다. 그러나 가장 위험한 사람은 **아무것도 안 하는 사람**이다. 앱을 안 열고, 글을 안 쓰고, 버튼을 안 누르는 사람을 기존 시스템은 잡을 수 없다.

### 감지하는 신호

| 유형               | 예시                                      | 기존 시스템   | 시계열 감지기     |
| :----------------- | :---------------------------------------- | :------------ | :---------------- |
| **침묵**           | 7일 연속 미기록                           | 감지 불가     | 🚨 플래그         |
| **온도 급락 추세** | 78° → 65° → 52° → 45° → 38° (5일 연속)    | 오늘 38°만 봄 | 🚨 하락 추세 감지 |
| **시간대 이상**    | 평소 오후 9시 기록 → 새벽 1시 → 2시 → 3시 | 그냥 저장     | 🚨 수면 패턴 이상 |
| **빈도 변화**      | 매일 기록 → 주 2회 → 주 0회               | 감지 불가     | 🚨 사용 감소 추세 |

### 체크리스트 검증

| #   | 질문                      | 답변                                               |
| :-- | :------------------------ | :------------------------------------------------- |
| 1   | 텍스트 입력을 대체하는가? | **아니오.** 사용자는 여전히 글을 쓴다.             |
| 2   | 데이터 품질 유지/향상?    | **향상.** 단일 시점에 시간 축 추가.                |
| 3   | 얹는 것 / 대체하는 것?    | **얹는 것.**                                       |
| 4   | 없어도 기존 시스템 작동?  | 예.                                                |
| 5   | 새로운 능력?              | **예.** "침묵"과 "행동 변화 추세"를 감지하는 능력. |

### 이해관계자별 가치

| 이해관계자  | 가치                                                               |
| :---------- | :----------------------------------------------------------------- |
| 정신과의사  | "내원 사이 기록 빈도 변화를 보면 상태 악화를 선제적으로 파악 가능" |
| 우울증 환자 | "내가 말할 힘이 없어도 누군가 알아챈다"                            |
| 보호자      | "아이가 7일간 기록을 안 하면 알려준다"                             |
| 투자자      | "패시브 모니터링은 사용자 이탈 후에도 가치를 유지하는 기능"        |
| 정치인/장관 | "고독사 예방의 핵심 지표: 일상 활동 중단 감지"                     |

### 가치 사슬에서의 위치

```
사용자가 글을 쓴다 ← 건드리지 않음
       ↓
AI가 텍스트를 분석한다 ← 기존 그대로
       ↓
마음 온도 산출 ← 기존 그대로
       ↓
  ★ 시계열 분석 레이어 ★
  (온도 추세, 기록 빈도 변화, 시간대 패턴, 미기록 감지)
       ↓
위기 감지 정확도 향상
```

## 킥 #2: 언어 지문 (Linguistic Fingerprint)

### 한 줄 정의

> **"사용자가 '괜찮다'고 써도, 글쓰기 패턴이 변했으면 괜찮지 않은 것이다."**

### 해결하는 문제

기존 AI는 사용자가 쓴 **"무엇(What)"**만 읽는다. 위험 키워드가 있으면 감지하고, 긍정 단어가 많으면 안정으로 판정한다. 하지만 사용자가 의식적으로 "괜찮아요"라고 쓰면 AI가 속는다. **내용을 조작할 수 있지만, 글쓰기 구조는 무의식적이라 조작하기 어렵다.**

### 감지하는 신호

사람은 우울해지면 텍스트의 **구조적 특성**이 변한다 (임상 심리학 연구 기반):

| 언어 지표          | 정상 상태 (Baseline)     | 이상 감지              |
| :----------------- | :----------------------- | :--------------------- |
| **문장 길이**      | 평균 42자                | → 평균 8자로 축소 🚨   |
| **어휘 다양성**    | 감정 형용사 12종 사용    | → 2종으로 단조화 🚨    |
| **시제 분포**      | 미래 시제 30%            | → 미래 시제 0% 🚨      |
| **주어 패턴**      | "나는~" 1인칭 빈번       | → 주어 생략, 수동태 🚨 |
| **감정 표현 깊이** | "정말 짜증나고 억울했다" | → "별로였다" 🚨        |

### 체크리스트 검증

| #   | 질문                      | 답변                                                                        |
| :-- | :------------------------ | :-------------------------------------------------------------------------- |
| 1   | 텍스트 입력을 대체하는가? | **아니오.** 오히려 텍스트가 많을수록 정확도 향상. 글쓰기를 더 장려.         |
| 2   | 데이터 품질 유지/향상?    | **향상.** "What" 분석에 "How" 차원 추가.                                    |
| 3   | 얹는 것 / 대체하는 것?    | **얹는 것.**                                                                |
| 4   | 없어도 기존 시스템 작동?  | 예.                                                                         |
| 5   | 새로운 능력?              | **예.** "거짓 괜찮음"을 감지. 의식적 내용 조작을 무의식적 구조 변화로 탐지. |

### 이해관계자별 가치

| 이해관계자  | 가치                                                                                       |
| :---------- | :----------------------------------------------------------------------------------------- |
| 정신과의사  | "환자의 축소 보고를 객관적으로 보정할 수 있는 최초의 도구"                                 |
| 상담사      | "한국어 '괜찮아요'의 다중 의미를 구조적으로 구분할 수 있다"                                |
| 우울증 환자 | "내가 의식하지 못하는 변화까지 알아채준다"                                                 |
| 투자자      | "우울증 언어 마커는 국제 학술 논문으로 입증된 방법론. On-Device 개인화 모델은 경쟁사 불가" |
| 보호자      | "아이가 괜찮은 척 써도, 패턴이 다르면 알 수 있다"                                          |
| 장관        | "NLP 기반 우울증 조기 발견은 파일럿에서 정량적 데이터를 생산할 수 있다"                    |

### On-Device 시너지

```
클라우드 기반 서비스:
  → 모든 사용자의 평균적 우울 패턴만 감지 가능
  → 개인화 불가

마음온 (On-Device):
  → 이 사람만의 "정상 상태" 언어 패턴(Baseline)을 학습
  → 이 사람만의 이상 징후를 감지
  → 개인화된 언어 지문 = 폰 밖으로 절대 유출 안 됨
```

### 가치 사슬에서의 위치

```
사용자가 글을 쓴다 ← 건드리지 않음. 오히려 강화됨.
       ↓
AI가 텍스트를 분석한다
  ├─ 기존: 감정 키워드, 위험 단어 (What)
  └─ ★ 신규: 문장 구조, 어휘 다양성, 시제, 주어 패턴 (How) ★
       ↓
마음 온도 산출 (기존 + 언어 지문 보정)
       ↓
의료진 대시보드
  └─ ★ "환자가 '괜찮다'고 했으나 언어 패턴 이상 감지" 플래그 ★
       ↓
알림 및 개입
```

## 킥 #3: 관계 지형도 (Relational Topology)

### 한 줄 정의

> **"사용자의 일기에서 '누구'가 등장하고, 그 사람과 연결된 감정이 어떻게 변하는지 읽는다."**

### 해결하는 문제

기존 AI는 사용자의 **감정 자체**만 본다. "슬프다", "화났다"를 감지할 수 있지만, **왜 슬프고 누구 때문에 화났는지**는 분석하지 못한다. 그리고 우울증의 가장 강력한 행동 지표 중 하나인 **사회적 위축(Social Withdrawal)** — 일기에서 타인이 서서히 사라지는 현상 — 을 감지할 수단이 없다.

### 감지하는 신호

사용자의 일기에서 AI가 자동으로 추출하는 것:

```
[팀장]  ──── 😰 위축, 😡 분노    ── 빈도: 주 3회 (증가 추세)
[엄마]  ──── 😢 죄책감, 💛 감사   ── 빈도: 주 1회 (안정)
[민수]  ──── 😊 즐거움            ── 빈도: 월 1회 → 0회 (감소 🚨)
[전체]  ──── 등장 인물 수: 3명 → 1명 → 0명 (사회적 위축 🚨)
```

우울증 진행 단계와 관계 지형의 변화:

| 단계          | 일기 속 대인관계 양상                   | 의미                  |
| :------------ | :-------------------------------------- | :-------------------- |
| **정상**      | 다양한 사람 + 다양한 감정               | 건강한 사회적 관계    |
| **초기 우울** | 부정적 관계만 남음, 긍정적 관계 소멸    | 선택적 사회 회피 시작 |
| **심화 우울** | 타인 자체가 사라짐. "나 혼자", "아무도" | 사회적 고립 = 고위험  |

**기존 시스템은 이걸 감지할 수 없다.** "오늘 혼자 있었다"에 위험 키워드는 없기 때문이다.

### 체크리스트 검증

| #   | 질문                      | 답변                                                                                            |
| :-- | :------------------------ | :---------------------------------------------------------------------------------------------- |
| 1   | 텍스트 입력을 대체하는가? | **아니오.** 사람 이름과 상황이 텍스트에 나와야 분석 가능. 풍부한 서술을 장려.                   |
| 2   | 데이터 품질 유지/향상?    | **향상.** 감정의 "대상(Object)"이라는 새 차원 추가.                                             |
| 3   | 얹는 것 / 대체하는 것?    | **얹는 것.** 기존 감정 분석 + 관계 엔티티 추출 레이어.                                          |
| 4   | 없어도 기존 시스템 작동?  | 예.                                                                                             |
| 5   | 새로운 능력?              | **예.** 현재는 "사용자의 감정" 자체만 봄. 이것은 "감정의 사회적 맥락"과 "대인관계 변화"를 감지. |

### 이해관계자별 가치

| 이해관계자  | 가치                                                                                                  |
| :---------- | :---------------------------------------------------------------------------------------------------- |
| 정신과의사  | **"이건 진료에 바로 쓴다."** 대인관계 패턴은 정신과 진료 핵심 질문. 2주치 데이터를 진료 전 미리 확인. |
| 상담사      | **"사례 개념화(Case Formulation)의 자동화."** 내담자의 관계 패턴을 시각적으로 파악.                   |
| 우울증 환자 | AI가 "지난주에 민수 이야기가 없네요"라고 물어보면 **"이 AI는 내 일상을 기억한다"**고 느낌.            |
| 투자자      | **"사회적 고립 → 고독사 예방"** B2G 스토리 완성. 정량 지표: "등장 인물 수 추이".                      |
| 정치인      | **"고독사 예방 = 사회적 관계 단절 감지."** 이 기능이 바로 그것.                                       |
| 장관        | **"기존 MHIS에서 추적 못 하는 '사회적 고립' 지표를 AI가 자동 생성."**                                 |

### 가치 사슬에서의 위치

```
사용자가 글을 쓴다 ← 건드리지 않음. 풍부한 서술 장려.
       ↓
AI가 텍스트를 분석한다
  ├─ 기존: 감정 키워드, 위험 단어 (What)
  ├─ 킥 #2: 문장 구조, 어휘 다양성 (How)
  └─ ★ 킥 #3: 등장 인물, 관계별 감정, 사회적 밀도 (Who) ★
       ↓
마음 온도 산출 (기존 + 관계 지형 보정)
       ↓
의료진 대시보드
  └─ ★ "환자의 사회적 관계 밀도 급감" 플래그 ★
       ↓
알림 및 개입
```

## 전체 종합: 마음온이 하나의 텍스트에서 읽는 4가지 차원

```
사용자의 일기 텍스트 (하나의 입력)
         │
         ├─ [기존] What: 무슨 말을 했는가 (키워드, 감정어)
         │
         ├─ [킥 #1] When: 언제, 얼마나 자주 쓰는가 (시계열)
         │
         ├─ [킥 #2] How: 어떻게 쓰고 있는가 (언어 지문)
         │
         └─ [킥 #3] Who: 누구에 대해, 관계가 어떻게 변하는가 (관계 지형도)
```

> **입력은 하나. 인사이트는 넷. 사용자는 그저 일기를 쓸 뿐이다.**

### 각 킥이 잡는 위기 유형

| 위기 유형                 | 기존           | 킥 #1             | 킥 #2               | 킥 #3               |
| :------------------------ | :------------- | :---------------- | :------------------ | :------------------ |
| "죽고 싶다" (명시적 위기) | ✅ 키워드 감지 | —                 | —                   | —                   |
| 기록 중단 (침묵)          | ❌             | ✅ 미기록 패턴    | —                   | —                   |
| "괜찮아요" (거짓 괜찮음)  | ❌ 속음        | —                 | ✅ 언어 구조 변화   | —                   |
| 사회적 고립 (관계 소멸)   | ❌             | —                 | —                   | ✅ 등장 인물 감소   |
| 수면 패턴 이상            | ❌             | ✅ 새벽 기록 급증 | —                   | —                   |
| 감정 표현 건조화          | ❌             | —                 | ✅ 어휘 다양성 감소 | —                   |
| 특정 관계 악화            | ❌             | —                 | —                   | ✅ 관계별 감정 추이 |

> **기존 시스템이 잡는 건 "명시적 위기" 1가지뿐이다. 3개의 킥이 추가되면 7가지 유형의 위기를 감지할 수 있다.**
