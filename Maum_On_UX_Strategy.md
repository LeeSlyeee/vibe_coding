# Maum-On: On-Device AI 안심 UX 전략 (Security Strategy)

## 📌 핵심 가치 (Core Value)

> **"내 마음 속 이야기는 내 폰 속에만."**
> 마음온은 사용자의 내밀한 일기가 외부 서버로 전송되는 불안함을 원천 차단합니다. **On-Device AI** 기술을 통해 모든 상담과 분석이 사용자의 기기 내부에서 **독립적으로** 수행됩니다.

---

## 🛡️ 사용자 경험(UX) 시나리오별 전략

### 1. 온보딩 / 로딩 화면 (Onboarding)

앱 실행 시 모델을 메모리에 올리는 시간(2~3초) 동안, 단순히 '로딩 중'이라고 하기보다 보안성을 강조하여 신뢰를 형성합니다.

- **UI 요소**: 모델 로딩 프로그레스 바 + 안심 문구
- **메인 카피**: "회원님의 폰 안에 '나만의 상담실'을 짓고 있어요."
- **서브 카피**:
  > "🔒 마음온의 AI 상담사는 인터넷 연결 없이 회원님의 기기 안에서만 존재합니다. 어떤 이야기도 외부로 유출되지 않으니 안심하세요."

### 2. 일기 작성 화면 (Writing View)

사용자가 글을 쓰는 순간, 이 데이터가 안전하다는 것을 시각적으로 인지시켜야 합니다.

- **UI 요소**: 우측 상단 `Security Badge` (🟢 초록색 방패 아이콘)
- **상태 표시 (On-Device)**: `On-Device Secure`
- **툴팁 (Tooltip)**: "작성 내용은 서버로 전송되지 않고, 기기 내 AI가 직접 읽고 공감합니다."

### 3. 분석 보고서 화면 (Analysis View)

AI가 분석한 결과를 보여줄 때, 이것이 '어디서' 왔는지 명시합니다.

- **헤더 배지**: **"By Local Gemma 2"** 또는 **"🔒 Local AI Analysis"**
- **설명**: "이 분석 결과는 회원님의 아이폰이 직접 생각하고 작성했습니다."

---

## ☁️ 저사양 기기 (Cloud Mode) 대응 전략

RAM 6GB 미만 (iPhone 12, 11 등) 사용자의 경우, 부득이하게 서버 AI를 사용해야 하므로 **투명성**과 **데이터 관리**를 강조합니다.

- **배너 알림**: "더 깊은 공감을 위해 클라우드 AI 연결됨"
- **안심 문구**:
  > "기기 성능 한계로 서버의 도움을 받지만 걱정 마세요.
  > 전송된 일기는 **철저히 암호화**되며, 분석 즉시 **자동 파기**됩니다."
