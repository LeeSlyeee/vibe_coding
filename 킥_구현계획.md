## 아키텍처 개요

```
iOS 앱    ─┐                    ┌─ 킥 #1: 시계열 분석 (SQL 집계)
           ├─ (동기화) ─→ OCI ──┼─ 킥 #2: 언어 지문 (KoNLPy)
Android 앱 ─┘                    └─ 킥 #3: 관계 지형도 (KoNLPy NER)
                                        │
                                        ▼
                                분석 결과 → DB 저장
                                        │
                                        ▼
                            의료진 대시보드 / 보호자 알림
```

## Phase 1: 킥 #1 — 시계열 패턴 감지기

### 구현 범위

외부 라이브러리 **불필요**. 기존 Flask + SQLAlchemy만으로 구현.

### 파일 구조

```
backend/
├── kick_analysis/              ← 신규 디렉토리
│   ├── __init__.py
│   ├── timeseries.py           ← 킥 #1 분석 로직
│   ├── linguistic.py           ← 킥 #2 분석 로직 (Phase 2)
│   └── relational.py           ← 킥 #3 분석 로직 (Phase 3)
├── kick_routes.py              ← 킥 분석 API 엔드포인트
```

### 분석 항목

| 분석                 | 로직                                                 | 데이터 소스        |
| :------------------- | :--------------------------------------------------- | :----------------- |
| **미기록 감지**      | 마지막 기록일로부터 오늘까지 경과 일수 계산          | `Diary.date`       |
| **기록 빈도 변화**   | 최근 7일 기록 수 vs 이전 7일 기록 수 비교            | `Diary.date`       |
| **마음 온도 추세**   | 최근 5일 mood_level 이동 평균 계산, 연속 하락 감지   | `Diary.mood_level` |
| **기록 시간대 분석** | created_at의 시간대 분포, 새벽(0-5시) 기록 비율 계산 | `Diary.created_at` |

### 출력 형식 (JSON)

```json
{
  "user_id": 1,
  "analysis_date": "2026-03-03",
  "timeseries": {
    "days_since_last_entry": 0,
    "inactivity_flag": false,
    "recent_7d_count": 5,
    "prev_7d_count": 7,
    "frequency_change": -28.6,
    "mood_trend": {
      "values": [78, 65, 52, 45, 38],
      "direction": "declining",
      "consecutive_decline_days": 5,
      "decline_flag": true
    },
    "night_recording": {
      "night_ratio": 0.6,
      "night_flag": true
    }
  }
}
```

### 플래그 기준 (임상 문헌 기반)

| 플래그                | 조건                                      | 심각도        |
| :-------------------- | :---------------------------------------- | :------------ |
| `inactivity_flag`     | 미기록 7일 이상                           | 🔴 High       |
| `decline_flag`        | mood_level 3일 연속 하락 AND 현재값 ≤ 40  | 🟠 Medium     |
| `night_flag`          | 새벽(0~5시) 기록 비율 50% 이상 (최근 7일) | 🟡 Low-Medium |
| `frequency_drop_flag` | 기록 빈도 50% 이상 감소 (vs 이전 7일)     | 🟠 Medium     |

### 수정 대상 기존 파일

| 파일        | 변경 사항                    |
| :---------- | :--------------------------- |
| `app.py`    | kick_routes 블루프린트 등록  |
| `models.py` | 변경 없음 (기존 필드만 사용) |

### API 엔드포인트

```
GET /api/kick/timeseries/<user_id>     ← 특정 사용자의 시계열 분석 결과
GET /api/kick/timeseries/flags         ← 전체 사용자 중 플래그 발생자 목록 (대시보드용)
```

## Phase 2: 킥 #2 — 언어 지문 (KoNLPy 필요)

### 사전 조건

- OCI 서버에 KoNLPy + Mecab 설치 필요
- OCI 서버 아키텍처 확인 필요 (ARM vs x86)
- `pip install konlpy` + Mecab 한국어 사전 설치

### 분석 항목

| 분석               | 로직                                   | KoNLPy 필요? |
| :----------------- | :------------------------------------- | :----------- |
| **문장 길이 평균** | 문장 분리 → 길이 계산                  | ❌ 불필요    |
| **어휘 다양성**    | unique words / total words             | ❌ 불필요    |
| **시제 분포**      | 형태소 분석 → 과거/현재/미래 어미 비율 | ✅ 필요      |
| **대명사 빈도**    | 형태소 분석 → 1인칭 단수/복수 비율     | ✅ 필요      |
| **감정어 다양성**  | 감정 형용사 태깅 → unique 감정어 수    | ✅ 필요      |

### 분석 방식

```python
# 기본형 (KoNLPy 없이)
def basic_linguistic(text):
    sentences = re.split(r'[.!?\n]+', text)
    avg_len = sum(len(s.strip()) for s in sentences if s.strip()) / max(len(sentences), 1)
    words = text.split()
    diversity = len(set(words)) / max(len(words), 1)
    return {"avg_sentence_length": avg_len, "vocab_diversity": diversity}

# 고급형 (KoNLPy 필요)
```

### Baseline 구축

- 사용자의 최초 14일 기록을 "정상 상태(Baseline)"로 설정
- 이후 기록을 Baseline과 비교하여 변화율 계산
- Baseline은 user별로 별도 저장 (DB 또는 JSON 필드)

## Phase 3: 킥 #3 — 관계 지형도 (KoNLPy NER 필요)

### 사전 조건

- Phase 2의 KoNLPy 설치가 선행되어야 함
- 한국어 일기 특성상 고유명사 + 호칭 패턴 모두 처리 필요

### 분석 항목

| 분석                  | 로직                                        |
| :-------------------- | :------------------------------------------ |
| **인물 추출**         | NER + 호칭 사전 (엄마, 아빠, 팀장, 친구 등) |
| **인물별 감정**       | 인물 주변 감정어 태깅                       |
| **등장 인물 수 추이** | 주간 unique 인물 수 카운팅                  |
| **사회적 밀도 변화**  | 등장 인물 수의 시계열 추세                  |
